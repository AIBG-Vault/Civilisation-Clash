# Layout Options

Open each HTML file in a browser to preview. All use the same clean Apple-inspired style with blue/orange team colors. Each demonstrates a different **layout and information architecture**. Rectangular grid. Light/dark mode supported.

---

## 1. Three Column

[layout-columns.html](layout-columns.html)

Classic three-column layout: left panel for your team, center for the game board, right panel for opponent. Stats are always visible. Action buttons below your stats. Collapsible sections for unit list and action log. Board is the focal point.

---

## 2. Overlay HUD

[layout-overlay.html](layout-overlay.html)

Full-screen game board with floating HUD panels. Player stats in top corners, action bar pinned to bottom. Panels are semi-transparent and can be minimized. Maximum board visibility. Click tiles to see details in a slide-out inspector panel on the right.

---

## 3. Tabbed Sidebar

[layout-tabs.html](layout-tabs.html)

Two-column layout with game board on left, tabbed sidebar on right. Tabs: Overview (both teams), Units (list with filters), Actions (build/move queue), Log (turn history). Compact and information-dense. Good for smaller screens.

---

## 4. Split Horizontal

[layout-split.html](layout-split.html)

Board on top, control panel spanning full width below. Control panel has horizontal sections: Your Stats | Actions | Opponent Stats | Turn Log. Maximizes board width. Collapsible control panel for full-screen board view.

---

## 5. Modal Actions

[layout-modal.html](layout-modal.html)

Minimal chrome - just the board and tiny status bar. All actions (build, move, end turn) open as modal dialogs. Unit details shown on hover/click in popover. Clean and focused, but requires more clicks. Good for spectators or touch devices.
